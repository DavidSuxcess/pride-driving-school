## Critical

### C1 — Архитектура передачи Telegram-токена клиенту

**Файл**: `src/lib/telegram.ts:2-3`, `EnrollmentForm.tsx`
**Что**: Токен бота читается через `import.meta.env.VITE_TELEGRAM_BOT_TOKEN`. Любая переменная с префиксом `VITE_*` в Vite **запекается в JS-бандл** при сборке.
**Эксплойт**: Если при `npm run deploy` существует `.env` с реальным токеном, любой посетитель сайта откроет devtools → найдёт токен → получит полный контроль над ботом (читать заявки клиентов, рассылать сообщения, удалять чаты).
**Сейчас**: В развернутом `dist/assets/index-*.js` токена **нет**. Vite вырезал весь Telegram-код целиком, потому что при последней сборке `.env` отсутствовал и `import.meta.env.VITE_TELEGRAM_BOT_TOKEN === undefined` → ранний return → DCE.

**Это объясняет H2: форма не отправляет заявки на лайве.**

**Решение** (требует обсуждения):
1. **Серверный прокси** (рекомендую): Cloudflare Worker / Vercel Function / маленький Node-сервис принимает форму и сам шлёт в Telegram. Токен живёт только на сервере.
2. **Сторонние формы** (быстрее): Formspree, Web3Forms — форма отправляется им, они шлют уведомления.
3. **Telegram Login Widget + bot webhook**: сложнее.

Без backend `npm run deploy` напрямую с `.env` — это утечка токена в публичный CDN.

## High

### H1 — Форма заявки сломана на live

**Severity**: High (функциональный баг)
**Где**: https://davidsuxcess.github.io/pride-driving-school/ — нажать «Записаться на курс», заполнить, отправить → всегда «Произошла ошибка».
**Причина**: см. C1 — Vite вырезал sending-код, осталось только `console.error("Telegram credentials are missing")`.
**Связано с C1** — лечится тем же решением.

### H2 — HTML-инъекция в Telegram-сообщении ✅ FIXED

**Файл**: `src/lib/telegram.ts:48-56`
**Что было**:
```ts
return `<b>👤 Имя:</b> ${data.name}\n📱 <b>Телефон:</b> <a href="tel:${cleanPhone}">${data.phone}</a>`;
```
Сообщение шлётся с `parse_mode: "HTML"`. Имя и телефон не экранируются → пользователь может ввести `<a href="https://evil.com">тест</a>` в поле «Имя» и оператор увидит искажённое сообщение или кликнет по ссылке.
**Уровень риска**: ограниченный (пострадает только оператор бота, не другие посетители), но это полноценная инъекция.
**Фикс применён**: добавлен `escapeHtml()` для всех пользовательских полей.

### H3 — `<html lang="en">` при русском контенте ✅ FIXED

**Файл**: `index.html:2`
**Что**: SEO/доступность — скринридеры читают русский текст английским голосом. Поисковики могут понизить ранжирование.
**Фикс применён**: `lang="ru"`.

### H4 — npm vulnerabilities (high+critical)

**Файл**: `package-lock.json`
17 уязвимостей, из них:
- **Critical**: `form-data` — небезопасный random для boundary (затрагивает `node-telegram-bot-api`, т.е. бот; на frontend не влияет)
- **High**:
  - `vite 7.0-7.3.1` — Path Traversal в dev-сервере (только локально)
  - `rollup 4.0-4.58.0` — Arbitrary File Write (только при сборке)
  - `lodash` — Code Injection через `_.template` (транзитивная)
  - `minimatch`, `picomatch` — ReDoS (транзитивные)
  - `flatted` — Prototype Pollution / DoS

Все production-критичные относятся к dev-tools и боту, не к runtime фронта. Но советую пофиксить для дисциплины.
**Фикс применён**: `npm audit fix` (non-breaking only). Бот-зависимости тронуть не могу, потому что `node-telegram-bot-api` major-bump может сломать `bot.js`.

### H5 — Доступность модалок: нет Esc, focus trap, focus restoration

**Файлы**: `Modal.tsx`, фото-модалка в `Reviews.tsx`, `Instructors.tsx`, `Header.tsx` (мобильное меню)
**Что**: Esc не закрывает модал. Tab может уйти за пределы модала. После закрытия фокус не возвращается на кнопку, открывшую модал. Скринридер не получает корректные сигналы.
**Влияние**: блокер для пользователей с клавиатурной навигацией / скринридерами.
**Фикс**: НЕ применил автоматически — требует переписывания нескольких компонентов. На обсуждение.

**Файлы**: `Modal.tsx:14-23`, `Header.tsx:29-38`

**Фикс**: завести один общий хук `useScrollLock()` со счётчиком, или хотя бы проверять состояние других «локеров».

## Low / Info

### L1 — Theme FOUC

**Файл**: `Header.tsx:11-25`, `index.html`
Тема ставится в React state → до hydration на `<html>` нет класса `dark`. На медленных соединениях видна вспышка светлой темы перед переходом в тёмную.
**Фикс**: inline-скрипт в `<head>` `index.html`, читающий `localStorage.theme` и ставящий класс до загрузки React.

### L2 — Нет SEO meta-тегов

**Файл**: `index.html`
Нет `description`, `og:image`, `og:title`, `twitter:card`. При шаринге в мессенджерах превью будет пустым.

### L3 — Нет шрифта Russo One

**Файл**: `Hero.tsx:53` использует `font-russo`, но в `index.html` подключён только Inter. Браузер падает на дефолт.
**Фикс**: добавить `Russo One` к Google Fonts link.

### L4 — `extendedPhotos`: одинаковый alt у 15 элементов

**Файл**: `Reviews.tsx:307`
Все 15 копий имеют `alt="Курсант"`. Скринридер прочитает 15 раз. Минор.

### L5 — `ErrorBoundary` не аудирован

**Файл**: `ErrorBoundary.tsx` — не читал. Если ловит и молчит — баги в проде не дойдут до Sentry/логов.

### L6 — Иконка favicon
